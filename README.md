# better-issues

A premium issue tracker with best-in-market UI and UX, built with the [Better-T-Stack](https://github.com/AmanVarshney01/create-better-t-stack).

## Tech Stack

- **TanStack Start** — Vite-powered SSR app runtime
- **TanStack Router** — File-based routing and navigation
- **TanStack Query** — Client-side server-state management
- **Convex** — Reactive backend (self-hosted local dev)
- **Better Auth** — Authentication via Convex HTTP actions
- **TailwindCSS + shadcn/ui** — Styling and components
- **Turborepo** — Monorepo build orchestration
- **Oxlint + Oxfmt** — Linting and formatting
- **Husky** — Git hooks

## Prerequisites

- [Bun](https://bun.sh/) (package manager and runtime)
- [Docker](https://docs.docker.com/get-docker/) (required for local Convex backend)

## Getting Started

### 1. Install dependencies

```bash
bun install
```

### 2. Set up Convex (first time only)

Run the initial setup to configure the local Convex backend:

```bash
bun run dev:setup
```

This runs `convex dev --configure --until-success` and generates env files in `packages/backend/`.

### 3. Configure environment variables

After setup, you need env files in **two** locations. Create them if they don't exist.

**`packages/backend/.env.local`** (generated by `dev:setup`):

```env
CONVEX_DEPLOYMENT=anonymous:anonymous-better-issues
CONVEX_URL=http://127.0.0.1:3210
CONVEX_SITE_URL=http://127.0.0.1:3211
```

**`apps/web/.env.local`** (create manually):

```env
CONVEX_DEPLOYMENT=anonymous:anonymous-better-issues
CONVEX_URL=http://127.0.0.1:3210
CONVEX_SITE_URL=http://127.0.0.1:3211
NEXT_PUBLIC_CONVEX_URL=http://127.0.0.1:3210
NEXT_PUBLIC_CONVEX_SITE_URL=http://127.0.0.1:3211
```

> **Important:** `NEXT_PUBLIC_CONVEX_URL` must point to port **3210** (the sync/backend endpoint). `NEXT_PUBLIC_CONVEX_SITE_URL` must point to port **3211** (the HTTP actions endpoint used by auth). Mixing these up causes a "Client disconnected" loop.

### 4. Set Convex backend environment variables

The Convex backend needs two runtime env vars for authentication. Set them using the Convex CLI:

```bash
bunx convex env set SITE_URL http://localhost:3001
bunx convex env set BETTER_AUTH_SECRET <your-secret>
```

Replace `<your-secret>` with a random string (e.g. `openssl rand -hex 32`).

These are **not** in `.env` files — they live in the Convex deployment environment.

### 5. Start development servers

```bash
bun run dev
```

This starts both:

- **Convex local backend** on `http://127.0.0.1:3210` (internal) and `http://127.0.0.1:3211` (HTTP/site)
- **TanStack Start app** on `http://localhost:3001`

Open [http://localhost:3001](http://localhost:3001) in your browser.

## Troubleshooting

### "Client disconnected" loop in Convex logs

If you see repeated `[INFO] Client disconnected` messages every few seconds:

1. **Make sure the Convex backend is running.** Run `bun run dev:server` in a separate terminal to verify the backend starts without errors.
2. **Check `NEXT_PUBLIC_CONVEX_URL`** in `apps/web/.env.local` — it must point to `http://127.0.0.1:3210` (port `3210`, not `3211`).
3. **Check that `SITE_URL` and `BETTER_AUTH_SECRET`** are set in the Convex environment (step 4 above). Missing auth config can cause connection issues.
4. **Restart both servers** after any env changes — env vars are read at startup.

### Auth not working

- Verify `SITE_URL` matches your app URL exactly (e.g. `http://localhost:3001`, no trailing slash).
- Verify `BETTER_AUTH_SECRET` is set and non-empty.

## Docker Compose Deployment (Dokploy)

Environment variables required for `docker-compose.yml`:

| Variable                       | Required | Description                                      |
| ------------------------------ | -------- | ------------------------------------------------ |
| `INSTANCE_SECRET`              | Yes      | Convex instance secret                           |
| `DOMAIN`                       | Yes      | Convex public domain (e.g. `convex.example.com`) |
| `APP_DOMAIN`                   | Yes      | Web app domain (e.g. `issues.example.com`)       |
| `CONVEX_SELF_HOSTED_ADMIN_KEY` | Yes      | Admin key for the Convex CLI                     |
| `BETTER_AUTH_SECRET`           | Yes      | Better Auth secret                               |
| `SITE_URL`                     | Yes      | Public URL of the web app                        |
| `CONVEX_DASHBOARD_DOMAIN`      | No       | Only if exposing the dashboard                   |
| `INSTANCE_NAME`                | No       | Defaults to `convex-prod`                        |
| `DOKPLOY_NETWORK`              | No       | Defaults to `dokploy-network`                    |

```bash
docker compose up -d --build
```
